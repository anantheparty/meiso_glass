# 系统设计

> 状态：产品设计文档。定义目标与稳定边界，不定义 wire、字段或实现步骤。

## 目标

Meiso Glass 面向分体式 XR/AR 眼镜：把受重量、功耗和散热约束的穿戴端与外部计算设备分开，并通过开放契约形成可复用、可扩展的 SDK 与生态。

系统不绑定某一颗 SoC、图形 API、操作系统或无线载体。硬件可以更换，公开的应用语义不随之重写。

## 两个角色

| Role | Responsibility |
|---|---|
| `Edge` | 驱动设备、采集传感器、维护本地姿态，完成最终合成、光学校正和显示安全 |
| `Host` | 运行业务与重计算，消费 Edge 数据并声明应用希望呈现的内容 |

`Host` 可以是 Linux、macOS 或其他受支持的计算设备。Host Runtime、CLI 和调试工具只是 Host 的包装，不是新的系统角色。

## 不变量

- 每个同步事实只有一个 truth owner；另一端只持有副本。
- 双方按能力协商，不假定 Edge 拥有通用 GPU，也不静默舍弃不支持的内容。
- `Edge` 始终拥有最终显示时机、修正能力与安全降级；无法满足 plan 要求时必须拒绝，不能静默省略修正。
- 网络接收不能直接修改正在显示的状态；显示侧只读取完整、已激活的一代内容。
- 所有 queue、重组和离线行为有界。断线或 lease 到期后进入声明过的本地行为。

## Composition 边界

内容在哪里生成，与内容如何锚定在空间中是两个独立维度。

每个 layer 可以来自 `REMOTE_PROJECTION`：Host 生成投影内容，Edge 解码并完成最终修正；也可以来自 `EDGE_NATIVE`：Host 同步意图、状态与资源，Edge 本地生成内容。两种来源可以共存，不是设备的全局二选一模式。

无论来源如何，layer 都进入 Edge 的最终 compositor。具体使用 GPU、固定功能单元、display plane 或软件实现，不属于公开契约。

## 通信边界

Meiso 定义的是适合 XR 的应用交付语义：可丢弃观测、最终状态、有序事件、完整资源，以及独立的实时媒体。它们位于统一 LinkSession 之上，不等同于多条物理连接。

标准 Wi-Fi 与同链路 IP/UDP 是 reference carrier。raw Wi-Fi 只有在测量证明它能显著改善目标硬件的延迟、功耗或调度控制时才增加为可选 adapter；它不得改变上层契约。

## 当前范围

当前仓库只验证了目标能力的一小部分；已实现内容与证据以[开发板验证](validation.md)为准，未验证设计不能被描述为稳定接口。
