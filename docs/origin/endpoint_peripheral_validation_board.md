# i.MX8M Mini 端侧外设验证板方案 V0

目的：基于现有 iTOP i.MX8M Mini 开发板/核心板，通过飞线和必要的底板外设断开，验证 AR 眼镜端侧的低功耗外设组合、电源域、接口资源和调试方法。Mini 核心、DRAM、PMIC、启动存储、A53/M4 基础调试仍使用开发板；本板不是完整 Mini 载板，也不是完整端侧主板。

当前结论：

- 全彩摄像头（CSI）和显示模组（DSI）先放在开发板已有 MIPI 接口侧，不进入本外设验证板。
- 外设验证板 V0 只做一组 `HM0360 + GW1NZ-2`，用于验证低功耗世界相机/眼动相机共用的基础切片。
- 主接口按 1.8V 低功耗域设计；3.3V 只作为开发板兼容、调试或可选射频性能对比，不作为默认产品形态。
- `UART2` 保留 A53 调试串口，`UART4` 保留 M4 调试串口；外设板不依赖 Mini UART 做主链路。
- 资源不足时，先释放开发板被占用的原生接口，再考虑外设独立供电/独立调试；GPIO 模拟只适合低速调试和慢速控制，不适合图像、音频和射频数据面。
- 开发板 5V 可以用于早期低负载调试，但不建议作为外设板默认供电来源；外设板应保留独立 5V 输入，并和开发板共地。

  分流电阻 / 0R 旁路 / INA 焊盘位 / 测试点

电源策略：

- 电源名统一采用 `1V8_MAIN`、`1V2_CAM_DVDD`、`2V8_CAM_AVDD` 这类格式。
- `1V8_MAIN` 是默认主控侧 I/O 域，也是射频、低功耗蓝牙、惯性传感器、麦克风的默认域。
- `HM0360` 按 `AVDD=2.8V, DVDD=1.2V, IOVDD=1.8V` 设计。
- `GW1NZ-2` 选 LV 版本，核心电源按 1.1V/1.2V 可配置，I/O 组默认 1.8V。
- `LR2021` 默认 1.8V 供电，避免 3.3V I/O 转换；保留 3.3V 射频供电跳线只做功放/效率对比。
- `TPS7A02` 和 `TPS7A20` 都保留焊盘位，通过 0R 或跳线二选一装配，便于比较静态电流、噪声、压差和瞬态响应。
- 所有电流监测芯片都要有旁路，避免测量芯片本身影响低功耗结论。

## 5. 功耗预算与供电建议

关键器件电流参考：

| 负载 | 低功耗/待机场景 | 活动场景 | 备注 |
|---|---:|---:|---|
| `HM0360` | 监测模式约 0.18-0.26mA | VGA 60 帧约 8.6mA | 电流分布在 AVDD/DVDD/IOVDD，实际还受 MCLK、DVP 位宽和帧率影响。 |
| `GW1NZ-2` | 取决于配置和时钟门控 | 需按 FPGA 逻辑实测，V0 先按 10-50mA 级别预算 | FPGA 是视觉切片功耗不确定性最大的部分。 |
| `LR2021` | 睡眠约 nA 级，接收约 5.7mA | 发射约 12mA@+10dBm、27mA@+14dBm、105mA@+22dBm | 射频峰值电流决定 5V 输入和 1V8/3V3 支路的瞬态设计。 |
| `nRF54L15` | 取决于广播/连接间隔 | 先按 15-30mA 峰值预算 | 后续用实际固件和发射功率复测。 |
| `ICM-45686` | 低功耗模式更低 | 6 轴低噪声模式约 0.42mA | 对 5V 输入影响很小。 |
| 2 个 `T5838` | 睡眠约 1.6-18uA，AAD 约 40uA | 低功耗模式约 0.24mA，高质量模式约 0.62mA | 两个麦克风合计仍是小负载。 |
| 电源测量/调试器件 | 取决于装配 | 可能达到 mA 级 | 功耗基线测试时应旁路或不装。 |

供电判断：

- 低负载寄存器调试时，外设板 5V 输入大概率在几十 mA 以内，开发板 5V 可以临时使用。
- 图像切片、FPGA、低功耗蓝牙和 LR2021 接收同时工作时，外设板 5V 输入建议按 100-200mA 峰值预留。
- LR2021 以高功率发射、FPGA 同时工作、调试测量器件全装时，5V 输入建议按 300-500mA 峰值预留，并给输入电容和负载开关浪涌留余量。
- 开发板 5V 引脚/扩展口的可用电流不应默认按 500mA 以上使用；V0 板建议放独立 USB-C 或台式电源输入，保护和连接器按至少 5V/1A 设计。
- 外部 5V 和开发板共用信号时必须共地；若外设板和开发板都可能供电，应加入跳帽或理想二极管方案，避免 5V 互相反灌。

## 6. 从开发板/核心板引出的接口

推荐分三层取信号。

| 层级 | 来源 | 用途 | 电压域 | 评价 |
|---|---|---|---|---|
| A | 开发板现有 MIPI CSI/DSI 接口 | 全彩摄像头、显示 | 按开发板接口 | 保留在开发板侧，不进外设板。 |
| B | 核心板（SOM）原生信号飞线 | I2C4/ECSPI/GPIO/PDM 等 | 目标 1.8V，逐项实测 NVCC | 推荐路径，最接近低功耗产品形态。 |
| C | 开发板接口 | 临时调试/兼容 | 多数 UART/I2C/PDM/SAI 已转 3.3V；开发板 U60 接口为 1.8V | 只作为辅助，不作为默认主链路。 |

开发板 U60 是底板上的 `SDIO/GPIO/I2C4(1.8V)` 扩展接口。后文写“开发板 U60 接口”时，均指这个 1.8V 扩展接口，而不是芯片器件。

### 6.1 必选主控侧接口

| 信号组 | 建议来源 | 电压域 | 信号 | 用途 | 备注 |
|---|---|---|---|---|---|
| 电源 | 外部 5V 优先，开发板 5V 可选 | 5V | `5V_IN`, `GND` | 外设板输入 | 默认外设板自生成所有低压电源轨；开发板 5V 只作低负载调试。 |
| VIO 参考 | 开发板 U60 接口或核心板 1.8V | 1.8V | `VDD_1V8_REF` | 电平参考，不大电流取电 | 用于确认主控 I/O 域和逻辑分析仪参考电平。 |
| 控制 I2C | 开发板 U60 接口或核心板原生 `I2C4` | 1.8V | `I2C4_SCL`, `I2C4_SDA` | INA、惯性传感器、GPIO 扩展器、FPGA 低速寄存器 | 优先主控总线。 |
| SPI | 核心板原生 `ECSPI2` 或其它可飞线 SPI | 待实测，目标 1.8V | `SCLK`, `MOSI`, `MISO`, `CS0`, 额外 GPIO CS | FPGA/LR2021 高速控制或数据 | 若用 `ECSPI2`，必须先隔离底板 MCP2515/CAN。 |
| GPIO | 开发板 U60 接口 + 核心板 GPIO | 1.8V | `IRQ`, `RESET`, `EN`, `WAKE`, `BOOT/MODE`, `CSx` | 所有外设控制和中断 | V0 目标 8-12 根；不足时才启用 TCA6408A。 |
| PDM | 核心板原生 PDM/MICFIL | 按 NVCC 实测，目标 1.8V | `PDM_CLK`, `PDM_DATA0/1` | 2 个 T5838 麦克风 | 底板 J18 标为 3.3V，不建议直接接 1.8V 麦克风。 |
| 可选 UART | 核心板 `UART1` 或 `UART3` | 原生 1.8V / 底板 3.3V | `TX/RX` | 临时串口 | 不占用 `UART2` A53 调试和 `UART4` M4 调试。 |

### 6.2 本地调试接口

| 接口 | 信号 | 用途 | 是否依赖 Mini |
|---|---|---|---|
| FPGA JTAG | `TCK/TMS/TDI/TDO`, `PROGRAM/INIT/DONE` | GW1NZ 配置和 FPGA 逻辑上电调试 | 否 |
| nRF SWD | `SWDIO/SWCLK/RESET`, UART 日志可选 | 低功耗蓝牙固件和功耗调试 | 否 |
| LR2021 调试口 | SPI 排针 + `BUSY/DIO/RESET` | 可用 USB-SPI/逻辑分析仪独立调射频 | 否 |
| 电源测量接口 | 每个电源轨的跳线/分流电阻/测试点 | 单独供电、测流、断电 | 否 |
| 逻辑分析仪接口 | 主控 SPI/I2C/GPIO/PDM，旁边配 GND | 早期协议调试 | 否 |

## 7. 开发板外设断开/拆除建议

| 涉及资源/外设 | 建议处理 | 说明 |
|---|---|---|
| `UART2` / `UART4` | 保留 A53/M4 调试串口，不给外设板使用 | 这两路在底板上已经作为调试串口，上电调试期间价值最高。 |
| `ECSPI2` / MCP2515 CAN | 若 V0 需要 Mini SPI，拆/断 MCP2515 相关连接，至少隔离 `CS/SCLK/MOSI/MISO` | 底板上 `ECSPI2` 已接 CAN，不能和外设板 SPI 并挂。 |
| 底板 3.3V 扩展信号 | 对所有从底板 3.3V 排针引出的临时线加串阻、跳帽或隔离 | 防止 3.3V 侧电平转换器或外设反向影响 1.8V 低功耗板。 |
| `UART3` / RS485 | 默认不用；若必须使用，先断开 RS485 收发器路径 | `UART3` 底板侧已经给 RS485 使用且为 3.3V。 |
| CAN/RS485/LED/USB 集线器等底板外设 | 可拆供电 0R、断使能或拔模块 | 降低开发板本身噪声和不必要功耗，便于判断外设板问题。 |
| MIPI CSI/DSI | 不飞线高速 MIPI | 直接使用开发板现成接口，避免高速信号完整性变量。 |

上电前必须实测：

- `NVCC_ECSPI` 实际电压。
- 飞线 GPIO 所在 I/O 组的实际电压。
- 开发板 U60 接口的引脚定义、复用关系和可用 GPIO 数量。
- 底板 3.3V 电平转换器是否仍被其它外设上拉或驱动。

## 8. Mini 接口资源评估

| 资源 | 开发板现状 | V0 分配 | 是否够用 |
|---|---|---|---|
| MIPI CSI | 开发板已有接口 | 全彩摄像头 | 够。低功耗 HM0360 不走 Mini CSI。 |
| MIPI DSI | 开发板已有接口 | 显示模组 | 够。外设板不承接 DSI。 |
| `UART2` | A53 调试，经底板转 3.3V | 保留调试 | 不可占用。 |
| `UART4` | M4 调试，经底板转 3.3V | 保留调试 | 不可占用。 |
| `UART1` | 底板转 3.3V 到扩展 | 临时调试可选 | 可用但不推荐做主链路。 |
| `UART3` | 底板 RS485 | 默认不使用 | 若要用需断 RS485。 |
| `I2C2/3` | 底板转 3.3V | 不作为默认低功耗接口 | 可临时调试，需电平转换。 |
| `I2C4` | 开发板 U60 接口，1.8V | 主控制 I2C | 够用。 |
| `ECSPI2` | 底板接 MCP2515 CAN | 释放后给 FPGA/LR2021 | 够 V0，但有冲突，需要隔离 CAN 并实测电压。 |
| GPIO | 原生够，底板暴露有限 | 8-12 根飞线/开发板 U60 接口 | V0 够；不足时启用默认不装（DNP）`TCA6408A`。 |
| PDM/MICFIL | 核心板有，底板 J18 标 3.3V | 优先飞原生 1.8V | 取决于能否飞到原生 PDM I/O 组。 |
| USB/以太网 | 开发板已有 | 主控调试/网络 | 够，不进外设板。 |

结论：如果允许飞线并断开不需要的底板外设，Mini 资源对 V0 够用。紧张点主要是 `ECSPI2` 与 CAN 冲突、PDM 暴露电平、GPIO 数量和飞线可靠性。UART 资源确实紧，不应作为外设板主数据通道。

## 9. GPIO 模拟接口可行性

| 接口/信号 | GPIO 模拟是否可行 | 建议 |
|---|---|---|
| 慢速 I2C | 可行 | 可用于早期寄存器读写、地址扫描和临时调试；长期方案仍建议用硬件 I2C，减少时序抖动和处理器占用。 |
| 低速 SPI | 有条件可行 | 可用于 LR2021 或 FPGA 的低速寄存器访问；不建议承载图像数据、射频 FIFO 高吞吐或严格时序。 |
| UART | 只适合应急 | M4 侧低波特率软件串口可临时使用；Linux 侧 GPIO 模拟串口不稳定，不建议作为方案接口。 |
| 片选/复位/使能/模式脚 | 可行 | 这类慢速控制最适合 GPIO 或 GPIO 扩展器。 |
| 中断/唤醒 | 不建议模拟 | 中断和唤醒应直接接 Mini GPIO，避免额外延迟和漏唤醒。 |
| PDM 麦克风 | 不可行 | 需要稳定时钟和采样路径，应接 Mini MICFIL/SAI/PDM 或 FPGA，不能靠普通 GPIO 模拟。 |
| HM0360 DVP/MIPI | 不可行 | 图像数据进 FPGA；Mini GPIO 不承接相机数据。 |
| MCLK/高精度时钟 | 不建议 | 用 Mini 时钟输出、FPGA 或专用振荡器，不用软件翻转 GPIO。 |

资源不足时的处理顺序：

1. 先断开/拆除底板不需要的外设，释放原生 `ECSPI/I2C/GPIO/PDM`。
2. 仍不够时，让 `GW1NZ-2`、`LR2021`、`nRF54L15` 各自保留本地供电和本地调试，Mini 只接最小控制面。
3. 慢速 EN/RESET/MUX/LED/模式选择脚不够时，启用默认不装（DNP）`TCA6408A`。
4. 低速寄存器访问可临时用 GPIO 模拟 I2C/SPI，但只作为上电调试退路。
5. 只有必须接开发板 3.3V 口时，才装 `PCA9306`/`SN74AXC8T245`/`NTS0104` 等电平转换器。
6. 不把扩展芯片或 GPIO 模拟用于 SPI 数据面、PDM、相机数据、精确中断或低延迟唤醒路径。

## 10. V0 连接器建议

| 连接器 | 类型 | 引脚预算 | 内容 |
|---|---|---:|---|
| `J_HOST_RAW_1V8` | 2.54mm/1.27mm 排针或飞线焊盘 | 30-40 | 5V/GND/VIO_REF/I2C4/ECSPI/GPIO/PDM/可选 UART。 |
| `J_HM0360` | FPC 或模块接口 | 按 HM0360 模块 | AVDD/DVDD/IOVDD/GND/MCLK/I2C/DVP/XSLEEP/XSHDOWN。 |
| `J_FPGA_JTAG` | 2x5 或 Tag-Connect | 10 | GW1NZ JTAG/配置。 |
| `J_LR2021_RF` | u.FL/SMA + SPI 排针 | 射频 + 10 | 射频测试、独立 SPI 调试。 |
| `J_NRF_SWD` | 2x5 或 Tag-Connect | 10 | SWD、RESET、UART 日志、参考电压。 |
| `J_PDM_MIC` | 板载麦克风焊盘 + 调试焊盘 | 8-10 | 2 个 T5838 的 PDM 时钟/数据、麦克风供电、WAKE/THSEL、GND。 |

## 11. 上电调试顺序

1. 空板优先用外部 5V 上电，确认所有降压电源/LDO 输出、PG/EN、负载开关默认状态。
2. 只装电源测量和假负载，确认每个电源轨可单独开关、可旁路测量。
3. 装 `HM0360`，用本地电源和 I2C 读 ID，确认 `AVDD/DVDD/IOVDD` 时序。
4. 装 `GW1NZ-2`，JTAG 下载最小位流，抓 HM0360 DVP 时序。
5. 通过 `I2C4`/SPI 飞线让 Mini 读 FPGA 寄存器/FIFO。
6. 装 `LR2021`，先用本地 SPI 工具发包，再接 Mini/M4 调度。
7. 装 `nRF54L15`，用 SWD 独立验证低功耗蓝牙空闲、连接、唤醒，再接 Mini 控制。
8. 装 `ICM-45686` 和 2 个 `T5838`，分别验证唤醒、PDM、M4/Linux 侧数据路径。
9. 最后再合并低功耗场景：A53 睡眠、M4 控制、FPGA+HM0360 低帧率、LR2021 突发发送、低功耗蓝牙命令接收。

## 12. 已评估与仍需实测

已可判断：

| 项目 | 判断 |
|---|---|
| 全彩摄像头和显示 | 放开发板现有 CSI/DSI 侧更合理，外设验证板不承接高速 MIPI。 |
| `HM0360 + GW1NZ-2` 数量 | V0 做一组可行，足够验证眼动/低功耗世界相机共用的基础切片。 |
| `UART2` / `UART4` | 应保留给 A53/M4 调试，不应分配给外设板。 |
| 底板 `UART1/3`、`I2C2/3` | 多数已转 3.3V，不适合作为默认 1.8V 低功耗接口。 |
| `ECSPI2` | 与底板 MCP2515 CAN 冲突；若作为外设板 SPI，必须隔离 CAN。 |
| GPIO 模拟 | 只适合低速 I2C/SPI 调试和慢速控制，不适合 PDM、相机数据、精确中断和低延迟唤醒。 |
| 麦克风数量 | V0 固定 2 个 T5838 即可，不再预留 4 麦位置。 |
| 外设板供电 | 开发板 5V 不建议作为默认供电；外设板应自带独立 5V 输入和输入电流测量。 |

仍需实测或落板确认：

| 检查项 | 影响 |
|---|---|
| `NVCC_ECSPI` 实际电压 | 决定 `ECSPI2` 是否可 1.8V 直连外设板。 |
| MCP2515/CAN 的物理隔离方式 | 决定是否拆芯片、断 0R、切线或只禁用片选。 |
| PDM/MICFIL 能否取到原生 1.8V I/O 组 | 决定 T5838 是否直连 Mini，或先经 FPGA/本地音频路径调试。 |
| 开发板 U60 接口的完整引脚复用 | 决定可用 GPIO 数量和是否需要启用 `TCA6408A`。 |
| GW1NZ-2 最终封装引脚分配 | 决定 HM0360 采用 8 位还是 4 位并口，以及调试 GPIO 留量。 |
| LR2021 1.8V 与 3.3V 发射性能差异 | 决定产品目标是否坚持 1.8V 射频，或局部引入 3.3V 射频电源轨。 |
| 开发板 5V 实际供电余量 | 决定是否允许从开发板取电做小场景调试；正式 V0 默认外部 5V。 |
| 飞线机械可靠性 | 决定是否需要做小 SOM 夹层/转接板。 |
